GlobX

Lightweight & powerful state management for Redux. 90% less code to write in compare to Redux/MobX.

Get Started

Lightweight

The source-code is just several lines of code. And exposing only one function. the newStore function. The store has only 1 function, and one hook.

Powerful

You can access to every part of the state or modify the state in-place everywhere. Just modify the state like and other JSON object, and call the function updateState it will rerender all the components that depended on the change.

Good for agilitiy.

No boilerplate code. Just call the function newStore with you initial state. No selectors, action builders, providers or other things. Just access the store and modify it wherever you need.

Mutable

You can use any JS function directly on the store: Array push, pull, and others. No need for cloning.

It working fast

Only the components that should be render are automatically rerender. You can even memoize all your components, they will rerender when needed. GlobX automatically render only once per rendering cycle, so you can updateStore as many times you want.

Fully support by IDEs

You get auto-complete out-of-the-box to all your store, keys and values. No need to create definition files, it automatically caclulated from your initial store. All smart language featurs like "Find all references", and "Rename Symbol" are working for all keys in the store, so it's make life easier when refactoring.

Fast renders.

To tell Globx that component is depened on specific path in the store you have to call the function useRerenderIfNeed and give it the path (it auto-completed).

You already know it

That it. Three functions. If you read until here, you know everything about GlobX. Just see the examle todo app how to use it.
